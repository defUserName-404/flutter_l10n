# Flutter L10n

> **Android Studio / IntelliJ IDEA plugin** — Extract raw strings from Dart files into ARB
> localization files in one keystroke, with full Riverpod and BuildContext awareness.

---

## What it does

Open any Dart file, press **`Ctrl+Shift+L`** (or `Cmd+Shift+L` on macOS), and the plugin:

1. Parses the file using IntelliJ's **Dart PSI** (not regex) to find every raw string literal
2. Shows a **review dialog** — edit keys, uncheck strings you want to skip
3. Writes entries to **`assets/l10n/app_en.arb`** (and any other locale ARB files you have)
4. Replaces each raw string with the correct **localized call**, chosen per-scope:

   | Scope detected at the string's location | Replacement |
   |---|---|
   | `WidgetRef ref` in scope (ConsumerWidget, Consumer builder) | `ref.watch(l10nProvider).key` |
   | Notifier / AsyncNotifier class | `ref.watch(l10nProvider).key` |
   | `BuildContext context` in scope | `context.l10n.key` |
   | None of the above | `AppLocalizations.of(context)!.key` |

5. **Auto-imports** only what the file actually needs
6. Runs **`flutter gen-l10n`** in the background and shows a balloon notification

## Installation

### Option A — Install from disk (recommended while in development)

```bash
# Clone and build
git clone https://github.com/defusername/flutter-l10n.git
cd flutter-l10n
./gradlew buildPlugin

# Output: build/distributions/flutter-l10n-1.0.0.zip
```

Then in Android Studio / IntelliJ IDEA:

```
Settings → Plugins → ⚙️ gear icon → Install Plugin from Disk…
```

Select `build/distributions/flutter-l10n-1.0.0.zip` and restart the IDE.

### Option B — Run in a sandbox IDE (during development)

```bash
./gradlew runIde
```

This launches a fresh IDE instance with the plugin already installed — no zip needed.

## Flutter project setup

The plugin is built around the following `l10n.yaml` configuration. Place this file in your project root:

```yaml
arb-dir: assets/l10n
template-arb-file: app_en.arb
output-localization-file: app_localizations.dart
output-class: AppLocalizations
output-dir: lib/core/config/l10n
preferred-supported-locales:
  - en
synthetic-locale: false
nullable-getter: false
```

### Required files

```
assets/
  l10n/
    app_en.arb          ← plugin writes here
    app_ar.arb          ← optional: plugin writes TODOs here too

lib/
  core/config/l10n/
    app_localizations.dart        ← generated by flutter gen-l10n
    l10n_providers.dart           ← you create this (see below)
    l10n_extension.dart           ← auto-generated by plugin on first run
```

### `l10n_providers.dart` — Riverpod setup

Create this file once. It is **not** auto-generated (it requires `build_runner`):

```dart
import 'package:flutter/widgets.dart';
import 'package:flutter_riverpod/flutter_riverpod.dart';
import 'package:riverpod_annotation/riverpod_annotation.dart';
import 'app_localizations.dart';

part 'l10n_providers.g.dart';

@riverpod
class LocaleNotifier extends _$LocaleNotifier {
  @override
  Locale build() => const Locale('en');

  void setLocale(Locale locale) => state = locale;
}

@riverpod
Future<AppLocalizations> appLocalizations(Ref ref) async {
  final locale = ref.watch(localeNotifierProvider);
  return AppLocalizations.delegate.load(locale);
}

@riverpod
AppLocalizations l10n(Ref ref) {
  return ref.watch(appLocalizationsProvider).requireValue;
}
```

Run `build_runner` once after creating it:

```bash
dart run build_runner build --delete-conflicting-outputs
```

### `main.dart` — pre-load the locale

```dart
void main() async {
  WidgetsFlutterBinding.ensureInitialized();

  const defaultLocale = Locale('en');
  final initialL10n = await AppLocalizations.delegate.load(defaultLocale);

  runApp(
    ProviderScope(
      overrides: [
        appLocalizationsProvider.overrideWith((_) async => initialL10n),
      ],
      child: const MyApp(),
    ),
  );
}
```

## Usage

### Basic extraction

1. Open any `.dart` file
2. Press `Ctrl+Shift+L` / `Cmd+Shift+L`
   (or right-click anywhere in the editor → **Extract Strings to L10n**)
3. Review the table — edit keys if needed, uncheck anything you want to keep as-is
4. If you have multiple locale ARB files, fill in translations in the next dialog
5. Click **OK**

The plugin handles the rest.

### What gets extracted

The PSI parser finds `DartStringLiteralExpression` nodes and filters out:

- `import` / `export` / `part` directives
- Asset paths (`assets/`, `fonts/`, `.png`, `.svg`, `.otf`, …)
- Route strings (`'/home'`, `'/auth/login'`)
- Hex color codes (`'#FF5733'`)
- Package identifiers (`'package:flutter/...'`)
- Annotation arguments (`@JsonKey(name: 'foo')`)
- Strings already inside `AppLocalizations` or `.l10n.` calls
- Interpolated strings (contain `$variable` or `${expr}`)
- Pure snake_case identifiers (likely keys, not UI text)
- URLs, emails, base64 blobs

### What the replacement looks like

**Before:**

```dart
// In a ConsumerWidget
Text('Welcome back'),

// In a Notifier
state = AsyncError(Exception('Network error'), StackTrace.current);

// In a plain StatelessWidget
Text('Cancel'),
```

**After:**

```dart
// In a ConsumerWidget  → Riverpod ref detected
Text(ref.watch(l10nProvider).welcomeBack),

// In a Notifier  → Notifier field detected
state = AsyncError(Exception(ref.watch(l10nProvider).networkError), StackTrace.current);

// In a StatelessWidget  → BuildContext detected
Text(context.l10n.cancel),
```

### Switching locale at runtime

```dart
// Anywhere you have a ref
ref.read(localeNotifierProvider.notifier).setLocale(const Locale('ar'));
```

## How scope detection works

The plugin walks **up** the PSI tree from the position of each string literal:

```
DartStringLiteralExpression   ← the raw string
  └─ parent chain...
       ├─ DartMethodDeclaration
       │    └─ checks formalParameterList for "WidgetRef" or "BuildContext"
       ├─ DartClassDefinition
       │    └─ checks superclass for "Notifier", "AsyncNotifier", etc.
       └─ DartFunctionBody / DartFunctionExpression
            └─ checks sibling parameter list for "(context, ref, ...)"
```

Each string in a file is resolved independently — a single file can produce a mix of
`ref.watch(...)`, `context.l10n.`, and `AppLocalizations.of(context)!` replacements depending
on where each string appears.

## Caveats

- **Interpolated strings are skipped** — `'Hello $name'` is not extracted. Refactor these
  manually using ARB placeholders.
- **`flutter gen-l10n` must be on PATH** — the runner calls `which flutter` to locate the
  binary. If it fails, run `flutter gen-l10n` manually.
- **`l10nProvider.requireValue` assumes initialization** — if `appLocalizationsProvider` hasn't
  resolved yet, this throws. The `main.dart` pre-load pattern prevents this.
- **PSI requires the Dart plugin** — the Dart plugin must be installed and enabled in your IDE.
